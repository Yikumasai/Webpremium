## 📖 概要

Webpremiumは、スマートプリロード技術により**ゼロ遅延**のウェブブラウジング体験を実現する革新的なChrome拡張機能です。リンクにマウスをホバーすると、バックグラウンドのプリロードウィンドウでページを事前に開きます。実際にリンクをクリックすると、プリロードされたタブがメインウィンドウにシームレスに移動し、待ち時間を感じさせません。

### ✨ 主な特徴

- 🎯 **ゼロ遅延体験** - マウスホバーでプリロード、クリックで即座に開く
- 🪟 **プリロードウィンドウ技術** - 独立したウィンドウでプリロード、メインウィンドウに干渉しない
- 🔄 **スマートタブ管理** - 既に開いているタブを自動検出して移動
- 📊 **リアルタイム統計** - プリロード効果と節約時間を追跡
- 🎨 **モダンなUI** - ダークモード対応、シンプルで美しいインターフェース
- ⚙️ **高度なカスタマイズ** - 豊富な設定オプションで個人のニーズに対応

## 🎯 機能

### コア機能

#### 1. スマートプリロード
- **マウスホバートリガー** - リンクにマウスをホバーすると自動的にプリロード
- **調整可能な遅延時間** - 0-1000msのホバー遅延設定をサポート
- **近くのリンク予測** - マウス付近のリンクをスマートに識別してプリロード
- **プリロード数制御** - 最大同時プリロード数を設定可能（1-10個）

#### 2. プリロードモード
- **プリロードウィンドウモード（推奨）** - 独立した最小化ウィンドウでプリロード、完全なページ読み込み、メインウィンドウに干渉しない
- **iframeプリロードモード** - 軽量なプリロード方式、互換性が高い

#### 3. スマートタブ管理
- **重複タブ検出** - 同じURLのタブを自動検出
- **自動ジャンプ** - 既に開いているリンクをクリックすると、既存のタブに自動移動
- **シームレス移動** - プリロードタブをメインウィンドウにシームレスに移動
- **メモリ最適化** - 重複タブを減らし、メモリ使用量を削減

#### 4. ネットワーク認識
- **スマート検出** - ネットワーク状態を自動検出
- **適応戦略** - 低速ネットワーク時にプリロードを自動的に減少
- **データ節約** - 弱いネットワーク環境でのデータ浪費を回避

#### 5. ビジュアルインジケーター
- **プリロード状態表示** - リンクの横に小さな点でプリロード状態を表示
- **読み込み中アニメーション** - オレンジ色の点で読み込み中を表示
- **読み込み完了マーク** - 緑色の点でプリロード完了を表示

#### 6. サイトルール管理
- **カスタムルール** - 特定のサイトでプリロードを有効/無効化
- **ドメインレベル制御** - ドメイン単位でのプリロード制御
- **右クリックメニュー** - 現在のサイトのプリロード状態を素早く切り替え

#### 7. 統計と分析
- **プリロード回数** - 総プリロード回数を記録
- **ヒット率統計** - プリロードの有効利用率を計算
- **時間節約** - 節約した総時間を統計
- **セッション時間** - 現在のセッションの使用時間を表示

### ショートカットキー

- `Alt + P` - プリロード機能の素早いオン/オフ
- `Alt + C` - すべてのプリロードキャッシュをクリア

### 右クリックメニュー

- **このリンクをプリロード** - 選択したリンクを手動でプリロード
- **このサイトでプリロードを有効/無効化** - 現在のサイトのプリロード状態を素早く切り替え

## 📦 インストール方法

### 方法1：デベロッパーモードでインストール

1. **ソースコードをダウンロード**
   [release](https://github.com/Yikumasai/Webpremium/releases)ページからダウンロード
   
   または
   
   ```bash
   git clone https://github.com/Yikumasai/webpremium.git
   ```

2. **Chrome拡張機能ページを開く**
   - アドレスバーに `chrome://extensions/` を入力
   - またはメニュー → その他のツール → 拡張機能

3. **デベロッパーモードを有効化**
   - ページ右上の「デベロッパーモード」スイッチをオン

4. **拡張機能を読み込む**
   - 「パッケージ化されていない拡張機能を読み込む」をクリック
   - ダウンロードした `webpremium` フォルダを選択

5. **インストール完了**
   - 拡張機能アイコンがブラウザツールバーに表示されます
   - アイコンをクリックして設定パネルを開きます

## 🎮 使い方

### 基本的な使い方

1. **拡張機能を有効化**
   - インストール後、拡張機能はデフォルトで有効
   - ツールバーアイコンをクリックして状態を確認

2. **プリロードを体験**
   - 任意のリンクにマウスをホバー
   - 設定した遅延時間を待つ（デフォルト100ms）
   - リンクの横に緑色の点が表示されたらプリロード完了
   - リンクをクリックすると即座に開きます

3. **統計を確認**
   - 拡張機能アイコンをクリック
   - 「統計」タブに切り替え
   - プリロード効果と節約時間を確認

### 詳細設定

#### ホバー遅延の調整
- 設定パネルを開く
- 「ホバー遅延」スライダーをドラッグ
- 推奨値：100-300ms

#### プリロード数の設定
- 設定パネルを開く
- 「最大プリロード数」スライダーをドラッグ
- 推奨値：3-5個

#### プリロードモードの選択
- **プリロードウィンドウモード**：完全なプリロード、最高の体験（推奨）
- **iframeモード**：軽量、互換性が高い

#### サイトルール管理
1. 「サイトルール」タブに切り替え
2. 「ルールを追加」ボタンをクリック
3. ドメインを入力（例：example.com）
4. 有効/無効状態を設定

### プリロードフロー

```
ユーザーがリンクにホバー
    ↓

    ↓
ネットワーク状態をチェック
    ↓
サイトルールをチェック
    ↓
プリロードウィンドウを作成
    ↓
プリロードウィンドウでタブを開く
    ↓
プリロードウィンドウを最小化
    ↓
ユーザーがリンクをクリック
    ↓
タブをメインウィンドウに移動
    ↓
タブをアクティブ化
    ↓
完了！
```

### 技術アーキテクチャ

- **Content Script** - ページのリンクイベントを監視、プリロードをトリガー
- **Background Service Worker** - プリロードウィンドウとタブを管理
- **Popup UI** - 設定インターフェースと統計情報を提供
- **Chrome Storage API** - 設定と統計データを永続化

### プリロードウィンドウ技術

拡張機能は独立したプリロードウィンドウを使用してページをプリロードします：

1. 小さなnormalタイプのウィンドウを作成
2. すぐにそのウィンドウを最小化
3. ウィンドウ内にプリロードタブを作成
4. ユーザーがクリックしたらタブをメインウィンドウに移動
5. タブをアクティブ化してメインウィンドウにフォーカス

この方式の利点：
- ✅ ページを完全にプリロード（JavaScript、CSS、画像など）
- ✅ メインウィンドウに全く影響しない
- ✅ タブをシームレスに移動可能
- ✅ すべてのサイトと複雑なページをサポート

## ❓ よくある質問

### Q: プリロードは多くのデータを消費しますか？
A: 拡張機能はネットワーク状態をスマートに検出し、低速ネットワークでは自動的にプリロードを減らします。「最大プリロード数」を調整してデータ消費を制御することもできます。

### Q: プリロードはブラウザのパフォーマンスに影響しますか？
A: プリロードは独立したウィンドウを使用するため、メインウィンドウのパフォーマンスへの影響は最小限です。また、拡張機能は期限切れのプリロードコンテンツを自動的にクリアします。

### Q: なぜ一部のサイトでプリロードが失敗するのですか？
A: 一部のサイトには保護メカニズムがある場合があります。「サイトルール」でこれらのサイトのプリロードを無効化できます。

### Q: リンクがプリロードされたことをどうやって知ることができますか？
A: 「インジケーター表示」を有効にすると、プリロード完了したリンクの横に緑色の点が表示されます。

### Q: プリロードウィンドウは表示されますか？
A: いいえ。プリロードウィンドウはすぐに最小化され、ブラウジング体験に全く影響しません。

### Q: 特定のサイトでプリロードを無効化できますか？
A: はい。「サイトルール」タブでドメインルールを追加するか、ページを右クリックして「このサイトでプリロードを有効/無効化」を選択します。
